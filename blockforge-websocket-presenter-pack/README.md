# BlockForge Labs: WebSocket Architecture Presenter Pack

This pack contains the complete recording plan for the first BlockForge Labs
realtime systems project. It is architecture-first and assumes the audience
already understands programming fundamentals.

## Recommended episode title

**Designing a Production-Style Realtime Backend with WebSockets**

## Documents

- `01-episode-presenter-script.md`: complete natural-flow narration, screen
  direction, codebase stops, transitions, and technical framing.
- `02-slide-and-screen-map.md`: the visual sequence for recording and editing.
- `03-codebase-walkthrough-map.md`: what to open in the Go repository, what to
  point at, and what each file proves.
- `04-milestone-roadmap.md`: implementation milestones, objectives, work,
  demonstrations, tests, and definitions of done.
- `05-architecture-decisions.md`: explicit version-one boundaries and tradeoffs.
- `06-recording-checklist.md`: preparation and quality-control checklist.

## Intended format

The main episode is not a live-coding tutorial. It combines:

- architecture diagrams and mental models
- focused codebase references
- failure and reliability analysis
- an explanation of deliberate version-one constraints
- a roadmap from a single node to a distributed implementation

The codebase is evidence for the architecture. It is not presented line by line.
